# 메모리 관리

## 프로세스와 메모리
### 프로세스의 동작
- 프로그램 카운터를 참조하여 메모리로부터 수행될 명령을 읽어 CPU가 해당 명령을 수행
- 프로그램은 일반적으로 보조기억장치에 파일 단위로 저장, 프로세스가 실행될 때 해당 프로그램은 로드되어야 함, 운영 체제는 보조기억장치에서 필요한 부분을 메모리로 옮겨 실행
- 자주 사용하는 프로세스나 데이터는 메모리에 있는 프로세스를 반복해서 사용할 경우 더욱 빠르 캐시에 저장해서 사용

### 기억장치 계층 구조
기억 장치 계층 구조는 컴퓨터 시스템에서 데이터를 저장하고 검색하는 방식을 효율적으로 구성하기 위해 사용되는 계층적 모델  
CPU 레지스터 => 캐시 메모리 => 메모리 => 보조 기억장치
- CPU 레지스터: 가장 빠른 메모리, 가장 비쌈
- 캐시 메모리: CPU 근처에 위치, 자주 사용되는 데이터를 저장해서 빠르게 접근하기 위해 사용
- 메모리: 프로그램 코드, 데이터, 스택 등을 저장하는 메모리로 운영 체제, 응용 프로그램 및 현재 실행 중인 프로세스와 관련된 데이터를 저장
  보조 기억장치: 하드 디스크, SSD 등과 같은 대용량 메모리

### 메모리 관리
- 메모리 호출: 새로운 프로세스가 생성되거나 실행될 때, 메모리 공간을 할당  
- 메모리 배치: 다음에 실행될 프로세스가 메모리에 어느 위치에 할당될지를 결정하는 과정  
- 메모리 교체: 메모리가 꽉 찬 상태에서 새로운 프로세스를 메모리에 적재해야 할 때, 운영 체제는 현재 메모리에 로드된 프로세스 중 일부를 선택하여 보조기억장치로 스왑  
- 고정/동적 분할  
- 고정/유동 적재영역

## 단일 프로그래밍 환경
초기 컴퓨터 시스템에서 사용했던 환경으로 하나의 프로그램만 실행되는 환경입니다.
- 오직 하나의 프로세스만 메모리를 사용
- 프로세스는 하나의 연속된 블록으로 메모리에 할당

**문제점**
- 메모리 용량을 초과하는 프로세스는 실행이 불가능합니다.
- 현재 실행중인 프로세스 외에 다른 프로세스는 사용할 수 없기에 메모리의 일부가 사용되지 않고 낭비가 됩니다.
- CPU나 I/O 디바이스를 사용하는 경우 반대쪽 자원이 사용되지 못하고 대기 상태에 있게 됩니다.

## 다중 프로그래밍 환경
운영체제가 여러 개의 프로세스를 동시에 메모리에 적재하고 실행하도록 허용하는 환경

메모리 분할은 여러 프로세스를 메모리에 효율적으로 적재하기 위한 관리 기법입니다. 크게 고정 분할 방식과, 동적 분할 방식으로 나뉩니다.  

**고정 분할 방식**  
고정 분할 방식은 메모리를 고정된 크기의 여러 개의 영역으로 나눕니다. 고정 분할 방식에는 프로세스 배치 방법이 2가지가 있습니다.

분할별 큐 방식
- 메모리를 고정된 크기의 영역으로 분할
- 프로세스의 각 분할된 영역에는 큐가 있고, 해당 큐에 들어온 프로세스는 해당 분할 영역에만 적재
- 절대 번역(한번 작업 큐에 들어간 작업을 다른 작업 큐로 옮겨 처리할 수 없음)과 적재를 사용
- 효율성이 낮음

단일 큐 방식
- 하나의 작업 큐를 사용하여, 큐에 들어온 프로세스는 어느 분할 영역에든 적재
- 재배치 가능 번역과 적재를 사용
- 복잡함

**동적 분할 방식**  
동적 분할은 메모리의 분할 경계가 고정되지 않고, 각 프로세스에 필요한 만큼의 메모리만 할당하는 메모리 관리 기법  
- 프로세스에 필요한 만큼의 메모리만 할당하기 때문에 메모리 활용도가 높음
- 프로세스의 크기와 메모리 공간의 크기가 일치하기 때문에 내부 조각화가 발생하지 않음
- 외부 조각화는 메모리 할당 및 해제 작업이 반복되면서 메모리 공간에 작은 크기의 빈 조각이 흩어져서 발생하는 현상

외부 조각화 해결 방법 ✔
통합: 인접된 공백을 더 큰 하나의 공백으로 만들어 외부 조각화를 해결
집약: 메모리 내의 모든 공백을 하나로 모아 외부 조각화를 해결

**메모리 보호**  
동적 분할 방식에서 프로세스가 다른 할당 영역을 침범하지 않도록 하기 위해 메모리 보호가 필요 각 프로세스에 하한 레지스터와 상한 레지스터를 할당하고, 프로세스는 하한 레지스터보다 큰 주소나 상한 레지스터보다 작은 주소에 접근할 수 없도록 합니다.

## 메모리 배치 기법
동적 분할 다중 프로그래밍 환경에서 새로 도착한 프로그램이나 데이터를 메모리에 어디에 배치할 것인지를 결정

1. 최초 적합  
  사용 가능한 공간 중에서 가장 먼저 발견되는 곳을 프로세스에 할당

3. 후속 적합  
  최초 적합 알고리즘의 변형으로, 이전에 탐색이 끝난 그 다음 부분부터 시작하여 사용 가능한 공간 중에서 가장 먼저 발견되는 곳을 프로세스에 할당

4. 최적 적합  
  필요한 공간을 제공할 수 있는 빈 공간 중 가장 작은 곳을 선택하여 프로세스에 할당  
  내부 조각화를 최소화하고 외부 조각화를 악화

5. 최악 적합  
  필요한 공간을 제공할 수 있는 빈 공간 중 가장 큰 곳을 선택하여 프로세스에 할당  
  외부 조각화를 최소화하고 내부 조각화를 악화
