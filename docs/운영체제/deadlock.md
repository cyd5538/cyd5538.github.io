# 교착상태
운영체제에서 교착 상태(Deadlock)는 둘 이상의 프로세스가 서로 필요한 자원을 점유하고 있어서 어느 프로세스도 진행될 수 없는 상태를 말합니다.

## 교착상태 필요조건
교착상태는 아래의 4가지 조건이 충족되어야 발생합니다.
1. 상호배제: 하나의 프로세스가 사용중인 자원은 다른 프로세스가 사용할 수 없습니다.
2. 점유대기: 프로세스가 이미 자원을 할당받은 상태에서 다른 자원을 기다리는 상태
3. 비선점: 이미 할당받은 자원을 강제로 빼앗아 사용할 수 없습니다.
4. 환형대기: 프로세스들이 서로 순환하는 방식으로 자원을 요구하는 경우.

## 자원할당 그래프
자원할당 그래프는 운영 체제에서 프로세스와 자원 간의 관계를 표현하는 그래프입니다. 자원할당 그래프를 사용하면 교착 상태를 분석하고 예방을 할 수 있습니다.

- 프로세스는 원이고 자원의 종류는 사각형입니다.

- 자원을 할당받았다면 자원에 점에서 원방향으로 화살표

## 교착상태 예방 방법

1. 상호배제 조건 제거
   - 공유할 수 있는 자원인 읽기 전용 파일 등은 상호배제 필요 없음
   - 공유할 수 있는 자원인 프린터 같은 경우 상호배제 필요
   - 프린터의 경우 여러 개의 가상 프린터로 나누어 각 프로세스가 독립적으로 사용하도록 설계

2. 점유대기 조건 제거
   - 프로세스가 실행하기 전에 필요한 모든 자원을 미리 할당받도록 설계
   - 프로세스가 대기중인 자원을 점유하지 않도록 설계

3. 비선점 조건 제거
   - 자원 할당을 선점 가능하도록 하거나, 대기상태가 방생하면 이미 할당된 자원을 모두 해제하여 다른 프로세스가 할당받을 수 있도록 설계

4. 환형대기 조건 제거
   - 모든 자원에 일련번호를 부여하고, 프로세스가 자원을 요구할 때 일련번호에 따라 요구 순서를 조절하여 환형대기를 방지

## 교착상태 회피
교착상태 회피는 운영체제에서 교착상태 발생을 사전에 방지하는 기법입니다.

교착상태 회피는 안전 상태라는 개념을 사용하여 작동합니다. 안전 상태는 모든 프로세스가 최대 자원량을 요청하더라도 교착상태에 빠지지 않는 상태를 의미합니다. 시스템은 자원 할당을 수행하기 전에 항상 안전 상태인지 확인하고, 안전 상태가 유지될 수 있는 방식으로만 자원을 할당합니다.

- 안전 상태: 모든 프로세스가 최대 자원량을 요청하더라도 교착상태에 빠지지 않는 상태(안전순서열이 존재하는 경우)
- 불안전 상태: 안전 상태가 아닌 상태. 교착상태가 발생할 가능성이 있는 상태(안전순서열이 존재하지 않는 경우)
- 안전 순서열: 안전 상태에서 시작하여 순차적으로 프로세스를 진행했을 때 교착상태가 발생하지 않는 프로세스들의 순서

1. 시스템은 모든 프로세스의 자원 사용에 대한 정보를 수집합니다.(현재 할당된 자원, 필요한 자원, 프로세스 최대 요구량)
2. 안전 상태인지 확인
3. 안전 상태가 아닌 경우 안전 순서열에 따라 프로세스를 진행
4. 프로세스가 자원을 사용한 경우 반납하고 다른 프로세스에 적용

### 교착 상태 회피 알고리즘
교착상태 발생을 사전에 방지하는 데 사용되는 알고리즘
1. 각 자원의 단위 자원이 하나밖에 없는 경우  
**변형된 자원 할당 그래프 이용**: 프로세스와 자원 간의 관계를 그래프로 표현하고, 그래프의 순환을 통해 교착상태 가능성을 검사
1. 각 자원의 단위 자원이 여러개인 경우
**은행원 알고리즘** : 프로세스가 필요로 하는 자원의 최대량과 현재 할당된 자원량을 기반으로 안전 상태를 판단하고 자원을 할당하는 알고리즘

## 교착상태 탐지 및 복구
교착상태를 발생한 후 탐지하고 해결하는 기법입니다.

- 교착상태 탐지
    - 주기적으로 상태 조사 알고리즘을 수행하여 교착상태 여부를 확인합니다
- 교차상태 복구
    - 교착상태가 탐지된 경우, 적절한 조치를 취하여 정상 상태로 복귀

**시스템이 교착상태에 있는지 교착상태를 탐지하는 shoshani와 coffman 알고리즘**
간단하고 효율적이지만 시간복잡도가 O(V2)로 높다.

**교차상태 복구**
- 교착상태를 일으킨 프로세스를 종료
    - 진행중이던 프로세스를 종료하면 데이터 손실 가능성도 있고 다시 복구가 어려움
- 교착상태를 일으킨 프로세스들의 자원을 취소
- 교착상태에 있는 프로세스를 차지하고 있는 자원을 다른 프로세스에 재할당
