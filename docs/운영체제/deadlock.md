# 교착상태
운영체제에서 교착 상태(Deadlock)는 둘 이상의 프로세스가 서로 필요한 자원을 점유하고 있어서 어느 프로세스도 진행될 수 없는 상태를 말합니다.

## 교착상태 필요조건
교착상태는 아래의 4가지 조건이 충족되어야 발생합니다.
1. 상호배제: 하나의 프로세스가 사용중인 자원은 다른 프로세스가 사용할 수 없습니다.
2. 점유대기: 프로세스가 이미 자원을 할당받은 상태에서 다른 자원을 기다리는 상태
3. 비선점: 이미 할당받은 자원을 강제로 빼앗아 사용할 수 없습니다.
4. 환형대기: 프로세스들이 서로 순환하는 방식으로 자원을 요구하는 경우.

## 자원할당 그래프
자원할당 그래프는 운영 체제에서 프로세스와 자원 간의 관계를 표현하는 그래프입니다. 자원할당 그래프를 사용하면 교착 상태를 분석하고 예방을 할 수 있습니다.

- 프로세스는 원이고 자원의 종류는 사각형입니다.
- 사용할 수 있는 자원은 사각형의 점으로 표시
- 자원을 할당받았다면 자원에 점에서 원방향으로 화살표
- 자원을 기다리고 있다면 원에서 자원에 점방향으로 화살표

## 교착상태 예방 방법

1. 상호배제 조건 제거
   - 공유할 수 있는 자원인 읽기 전용 파일 등은 상호배제 필요 없음
   - 공유할 수 있는 자원인 프린터 같은 경우 상호배제 필요
   - 프린터의 경우 여러 개의 가상 프린터로 나누어 각 프로세스가 독립적으로 사용하도록 설계

2. 점유대기 조건 제거
   - 프로세스가 실행하기 전에 필요한 모든 자원을 미리 할당받도록 설계
   - 프로세스가 대기중인 자원을 점유하지 않도록 설계

3. 비선점 조건 제거
   - 자원 할당을 선점 가능하도록 하거나, 대기상태가 방생하면 이미 할당된 자원을 모두 해제하여 다른 프로세스가 할당받을 수 있도록 설계

4. 환형대기 조건 제거
   - 모든 자원에 일련번호를 부여하고, 프로세스가 자원을 요구할 때 일련번호에 따라 요구 순서를 조절하여 환형대기를 방지
