# 프로세스

## 프로세스란?
프로세스는 컴퓨터 시스템에서 실행 중인 프로그램을 나타냅니다. 이러한 프로세스들은 운영체제로부터 자원을 할당받아 실행되며, 이 자원에는 CPU 시간, 메모리 공간, 입출력 장치, 파일 등이 포함됩니다. 프로세스는 운영체제의 프로세스 관리자에 의해 생성, 종료 및 관리됩니다.

**프로그램**: 동작을 하지 않는 정적-수동적 개체

**프로세스**: 동작을 하는 능동적 개체

프로세스는 운영체제로부터 자원을 할당 받아동작

- 자원: CPU, 메모리, 입출력장치, 파일 등
- 동작: CPU가 프로세스의 명령을 실행

운영체제(프로세스 관리자)가 처리하는 작업

- 프로세스를 생성 및 종료
- 프로세스를 실행시키기 위한 스케줄링 작업
- 프로세스의 상태 관리

## **프로세스의 구성**

### 메모리 구조

- 프로그램 실행에 직접적으로 필요한 코드와 데이터
  
![image](https://github.com/cyd5538/cyd5538.github.io/assets/91642972/53552f6b-28af-4ece-947c-3408fc3f06c0)  
[이미지 출처](https://zangzangs.tistory.com/107)

- **프로그램 코드(Code)**: 프로세스가 실행되는 명령어들로 이루어진 프로그램 코드가 포함됩니다. 이 코드는 메모리에 로드되어 CPU에 의해 실행됩니다.
- **데이터(Data)**: 프로세스가 실행 중에 사용하는 데이터가 포함됩니다. 이 데이터는 변수, 배열, 객체 등의 형태로 저장될 수 있으며, 프로세스가 실행되는 동안 변경될 수 있습니다.
- **스택(Stack)**: 프로세스의 지역 변수, 함수 호출 및 복귀 주소 등을 저장하는 메모리 영역입니다. 스택은 후입선출(LIFO, Last In First Out) 구조로 동작하며, 함수 호출 및 복귀에 사용됩니다.
- **힙(Heap)**: 동적으로 할당된 메모리 영역으로, 프로세스가 실행 중에 필요에 따라 메모리를 동적으로 할당하고 해제할 수 있습니다. 주로 동적으로 생성된 데이터나 객체들이 힙에 저장됩니다.

### 프로세스 제어 블록(Process Control Block: PCB)

- 운영체제가 프로세스를 관리하기 위해 필요한 정보
- 각 프로세스마다 존재
- 여러 프로세스가 번갈아 실행되는 경우 PCB에 저장된 정보 활용

![image](https://github.com/cyd5538/cyd5538.github.io/assets/91642972/475adfd4-898d-4cac-a101-cd33dea1fe9e)  
[이미지 출처](https://m.blog.naver.com/jk130694/220690136194)  

- **프로세스 상태(Process State)**:
    - 현재 프로세스의 상태를 나타냅니다. 예를 들어, 실행 중인지, 준비 중인지, 대기 중인지, 중지된 상태인지 등을 나타냅니다.
- **프로세스 식별자(Process ID)**:
    - 각 프로세스에 할당된 고유한 식별자입니다. 이를 통해 운영체제는 각 프로세스를 식별하고 구분합니다.
- **프로그램 카운터(Program Counter, PC)**:
    - 현재 실행 중인 명령어의 주소를 나타냅니다. 다음에 실행될 명령어의 주소를 가리키는 역할을 합니다.
- **레지스터 값(Register Values)**:
    - 프로세스가 현재 실행 중인 상태를 나타내기 위해 레지스터에 저장된 값들입니다. 이는 CPU 레지스터의 상태를 저장합니다. 레지스터 값에는 일반적으로 프로그램 카운터(PC), 스택 포인터(SP), 베이스 레지스터(BP) 등이 포함될 수 있습니다.
- **할당된 자원 정보(Allocated Resources)**:
    - 프로세스가 사용하는 자원들에 대한 정보를 포함합니다. 이는 CPU 시간, 메모리, 입출력 장치 등과 같은 자원들을 포함합니다. 이 정보를 통해 운영체제는 자원을 효율적으로 할당하고 관리합니다.
- **스케줄링 및 우선 순위 정보(Scheduling and Priority Information)**:
    - 프로세스의 실행 우선 순위나 스케줄링 정보를 나타냅니다. 운영체제는 이 정보를 기반으로 프로세스를 스케줄링하고 실행합니다.
- **프로세스의 상태 전이 정보(State Transition Information)**:
    - 프로세스가 상태를 변경할 때(예: 준비에서 실행으로, 실행에서 대기로), 이에 대한 정보가 포함됩니다. 이를 통해 운영체제는 프로세스의 상태 변화를 추적하고 제어합니다.

### 프로세스 생성과 종료

**프로세스 생성 방법**
- 프로세스는 여러 개의 새로운 프로세스를 새로운 프로세스를 생성할 수 있습니다.
- 자식 프로세스는 부모 프로세스의 복사본이 되거나, 부모와는 다른 프로세스를 실행할 수 있습니다.

**프로세스 생성 시스템 호출**

- UNIX, Linux: fork()
    - 자식 프로세스는 부모 프로세스의 복제본
- UNIX, Linux: exec()
    - 자식 프로세스는 부모 프로세스와는 다른 프로그램 실행
- Windows: CreateProcess()
    - 자식 프로세스는 새로운 프로그램 생성
 
**프로세스 종료 방법**
프로세스는 정상적 종료와 강제적 종료가 있습니다.
- 프로세스가 모든 처리를 완료(정상적 종료)
- 부모 프로세스에 의해 자식 프로세스를 강제로 종료
    - 프로세스 종료 시스템 호출 이용
    - 자식 프로세스 생성시 얻은 PID 이용
- 부모 프로세스가 종료되는 경우 운영체제가 자식 프로세스 종료
