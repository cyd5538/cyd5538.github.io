# 쓰레드

## 쓰레드란?
기존의 프로세스는 다음과 같은 특징을 가집니다.
- 하나의 프로그램을 실행하기 위한 기본적인 단위입니다.
- 자원 소유의 단위로, 하나의 메모리 구조를 가집니다.
- 디스패칭의 단위로, 하나의 제어 흐름을 갖습니다.
- 프로세스 내에서는 다중 처리가 불가능합니다.
  
프로세스 내에서의 다중 처리를 위해 쓰레드(Thread)라는 개념이 나타났습니다. 쓰레드(Thread)란 프로세스 내에서 실행되는 흐름의 단위를 의미합니다.
일반적으로 하나의 프로세스에는 하나의 쓰레드를 가지고 있습니다. 하나의 프로세스 안에 여러개의 쓰레드를 가지면 멀티 쓰레드라고 부릅니다.

## 쓰레드 특징

자원 효율성
- 프로세스보다 메모리 공간, CPU 시간, 시스템 오버헤드를 적게 사용합니다.
- 여러 작업을 동시에 처리하여 사용자 응답 속도를 향상시킬 수 있습니다.
- CPU 코어가 여러 개인 경우, 각 쓰레드를 각 코어에 할당하여 병렬 처리를 수행할 수 있습니다.

동시성 및 통신
- 여러 쓰레드가 동시에 실행될 수 있습니다.
- 프로세스 간 통신보다 쓰레드 간 통신이 더 효율적입니다.
- 쓰레드는 공유(데이터, 힙) 주소 공간을 사용하여 데이터를 교환하기에 간편하고 통신 비용이 감소합니다.

## 쓰레드 단점
- 하나의 스레드에서 발생한 문제가 프로세스 전반에 영향을 미칠 수 있습니다.
- 자원을 공유하기 때문에 동시에 여러 쓰레드가 자원에 접근할 때 동기화 문제가 발생할 수 있습니다.
