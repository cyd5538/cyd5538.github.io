JSX

JSX란 ?​

JSX는 JavaScript XML의 약자로, React에서 자바스크립트 코드를 HTML과 유사한 형태로 작성할 수 있도록 도와주는 문법 확장입니다. JSX는 브라우저에서 바로 실행되지 않으며, Babel과 같은 도구를 통해 일반 자바스크립트 코드로 변환된 뒤 실행됩니다.

장점​

더 쉬운 읽기 및 쓰기
HTML에 익숙한 개발자라면 JSX 문법을 더 쉽게 읽고 씁니다.

const hello = <h1>Hello, world!</h1>;

위 코드는 HTML처럼 보이지만 실제로는 React.createElement로 변환됩니다.

향상된 보안
새로운 엘리먼트를 생성할 수 있는 < 및 > 같은 위험한 문자가 HTML 문자열에 포함되었다면, JSX 코드를 컴파일할 때 다른 문자로 바꿔 더 안전한 자바스크립트 코드를 생성합니다. 앞서 말한 문자는 각각 HTML 문자의 부등호(&lt;와 &gt;)로 변환됩니다. 이 과정을 데이터 소독이라 합니다.

const userInput = "<script>alert('XSS!');</script>";
const safeElement = <div>{userInput}</div>;

위 코드에서 JSX는 <script> 태그를 단순 텍스트로 렌더링하여 보안을 강화합니다.

강력한 타이핑
JSX는 컴파일 단계에서 오류를 잡아내는 강력한 타입 검사를 지원합니다. 이를 통해 런타임 오류를 줄이고, 개발 초기 단계에서 문제를 해결할 수 있습니다.

컴포넌트 기반 아키텍쳐
JSX는 컴포넌트 기반 아키텍쳐를 권장하는데, 이는 코드를 보다 모듈화하고 유지 보수를 쉽게 하는데 도움을 줍니다.

const Button = ({ label }) => <button>{label}</button>;
const App = () => <Button label="Click Me" />;

광범위한 사용
JSX는 React에 국한되지 않고 Vue, Solid, Qwik 등 다른 라이브러리와 프레임워크에서도 사용됩니다.

약점​

학습 곡선 가중
JSX에 익숙하지 않은 개발자는 배우고 이해하기 어려울 수 있습니다.

전용 도구 필요
JSX는 브라우저에서 바로 실행되지 않으므로, Babel 또는 TypeScript 같은 트랜스파일러를 통해 변환해야 합니다.

관심사 혼합
JSX는 HTML과 자바스크립트 코드를 같은 파일에 작성하도록 권장합니다. 직관적이지만, 관심사의 분리를 선호하는 전통적인 개발자에게는 거부감을 줄 수 있습니다.

자바스크립트 호환 부족
JSX는 자바스크립트 문법과 유사하지만 완전히 동일하지는 않습니다. 예를 들어, JSX에서는 class 대신 className, for 대신 htmlFor를 사용해야 합니다.

const element = <div className="my-class"></div>;

JSX는 이러한 약점에도 웹 개발자, 특히 리액트를 사용하는 개발자들이 널리 애용하는 기술이 되었습니다. JSX를 사용하면 강력하고 유연한 방식으로 컴포넌트를 만들고 사용자 인터페이스를 작성할 수 있습니다. JSX는 동적이고 반응이 빠른 사용자 인터페이스를 작성하는 데 이로운 강력하고 유연한 도구입니다. JSX의 목적은 하나입니다. 반복, 계산, 인라인 실행 같은 강력한 기능을 그대로 유지하면서 리액트 컴포넌트의 코드를 간단하게 작성하고, 읽기 쉽게 만들고, 유지 보수하기 좋게 만드는것입니다.

내부 동작​

const a = 1;
let b = 2;

console.log(a+ b);

이 코드를 이해하고 나면 JSX를 더 잘 이해하게 되고, 리액트에 대한 이해도 깊어져 리액트를 능숙하게 다를 수 있게 될 것입니다.

어떻게 작동하나요?​

결론부터 말하면 컴파일러를 사용해 코드를 컴파일하는 것입니다. 컴파일러는 특정 규칙에 따라 고급 프로그래밍의 언어로 작성된 소스 코드를 구문 트리로 변환하는 소프트웨어입니다.

구문트리

문자 그대로 자바스크립트 객체 같은 트리 자료 구조입니다.

컴파일러는 3단계 과정을 거칩니다. 각각 토큰화, 구문 분석, 코드 생성이라고 하는데, 각 단계에 대해 자세히 알아보겠습니다.

토큰화
문자열을 의미 있는 토큰으로 분해하는 것입니다. 토크나이저가 상태를 가지고 있고, 각 토큰이 자신의 부모나 자식에 관한 상태를 포함하고 있을 때는 토크나이저를 렉서라고 부릅니다. 한마디로 렉싱은 상태를 가지는 토큰화입니다.

렉서는 렉서 규칙이 있어서, 이 규칙으로 정규 표현식을 사용해 프로그래밍 언어를 나타내는 텍스트 문자열에서 변수 이름, 객체 키 및 값 같은 주요 토큰을 감지합니다. 그런 다음 렉서는 구현에 따라 이러한 키워드를 열거 가능한 값으로 표현합니다. const는 0으로, let은 1로, functuon은 2로 바뀝니다.

문자열이 토큰화되거나 렉싱되면 다음 단계인 구문 분석으로 넘어갑니다.

구문 분석
토큰을 가져와 구문 트리로 변환하는 과정입니다. 구문 트리는 코드의 자료를 나타내는 자료구조입니다. 위에 있는 코드는 다음과 같은 구문 트리로 표현됩니다.

  "body": [

코드 생성
컴파일러가 추상 구문 트리(AST)에서 기계어를 생성하는 과정입니다. 여기에는 컴퓨터의 프로세서가 직접 실행할 수 있는 일련의 명렁어로 AST의 코드를 변환하는 작업이 포함됩니다. 이후, 결과로 생성된 기계어는 자바스크립트 엔진에 의해 실행됩니다. AST를 기계어로 변환하는 과정에는 복잡하고 다양한 단계가 포함됩니다. 하지만 최신 컴파일러는 매우 정교하며 다양한 하드웨어 아키텍쳐에서 효율적으로 실행되는 고도로 최적화된 코드를 생성할 수 있습니다.

웹 브라우저를 비롯해 최신 환경에서는 자바스크립트 코드를 효율적으로 실행하기 위해 JIT 컴파일러를 많이 사용합니다. 이러한 시스템에서는 자바스크립트 소스 코드가 먼저 바이트코드 같은 중간 표현으로 변환됩니다. 그런 다음 JIT 컴파일러는 프로그램이 실행될 때 이 바이트코드를 기계어로 동적 컴파일합니다. 이렇게 실행할 때 컴파일하면 엔진은 변수 종류와 자주 실행되는 코드 경로 같은 실시간 정보를 기반으로 최적화를 수행 할 수 있습니다.

런타임은 일반적으로 엔진과 연동해 특정 환경에 맞는 콘텍스트와 헬퍼 기능을 더 많이 제공합니다. 자바스크립트 런타임은 구글 크롬 같은 일반적인 웹 브라우저입니다. 크롬 웹 브라우저는 엔진과 연동하는 크로미움 런타임을 제공합니다 마찬가지로 서버 측에서는 V8 엔진을 사용하는 Node.js 런타임을 사용합니다.

런타임은 자바스크립트 엔진에 브라우저 런타임이 제공하는 window 객체와 document 객체 같은 콘텍스트를 제공합니다. Node.js에서는 전역 window 객체가 없음을 알 것입니다. 런타임이 달라서 다른 콘텍스트를 제공하기 때문입니다.

JSX로 자바스크립트 구문 확장하기​

전통적인 컴파일 과정에서는 소스 코드가 중간 단계를 거쳐 최종적으로 기계어로 변환됩니다. 그러나 JSX는 다른 접근 방식을 취합니다. JSX는 구문 트리(AST)를 통해 처리되어 모든 자바스크립트 엔진이 해석할 수 있는 표준 자바스크립트 코드로 변환됩니다.

이러한 변환 작업은 자바스크립트 생태계에서 바벨이 주로 담당하며, 타입스크립트, 트레이서, SWC와 같은 도구들도 유사한 역할을 수행합니다. 이러한 이유로 JSX는 브라우저에서 직접 실행될 수 없으며, 특수한 파서를 통해 구문 트리로 변환한 후 표준 자바스크립트로 컴파일하는 '빌드 단계'가 필요합니다.

이 과정을 트랜스파일이라고 부릅니다. 트랜스파일은 한 언어로 작성된 코드를 비슷한 수준의 추상화를 가진 다른 언어로 변환하는 것을 의미합니다. JSX의 경우, 리액트 요소를 생성하는 함수 호출로 변환되어 자바스크립트 런타임이 이해할 수 있는 형태가 됩니다.

JSX 프라그마​

JSX 프라그마는 < 기호로 시작하는 특별한 구문입니다. 자바스크립트에서 이 기호는 비교 연산자로 사용될 때를 제외하면 일반적으로 허용되지 않는 문법입니다. JSX에서는 이러한 XML/HTML과 유사한 태그 구문(JSX 프라그마)이 자바스크립트 함수 호출로 변환됩니다.

프라그마(pragma)라는 용어는 일반적으로 프로그래밍 언어의 표준 문법을 넘어서는 추가적인 정보나 지시사항을 컴파일러에 제공하는 특별한 지시어를 의미합니다. JSX의 경우, 이 프라그마 구문은 리액트나 다른 라이브러리에서 사용할 요소를 생성하는 함수 호출로 변환되도록 컴파일러에 지시합니다.

<MyComponent prop="속성값">콘텐츠 </MyComponent>

위 JSX 코드는 트랜스파일 과정을 거쳐 다음과 같은 자바스크립트 코드로 변환됩니다:

React.createElement(MyComponent, { prop: "속성값"}, "콘텐츠");

이러한 변환 메커니즘은 본질적으로 복잡한 UI 구조를 선언적으로 표현할 수 있게 해주는 문법적 설탕(syntactic sugar)입니다. 개발자는 중첩된 함수 호출 대신 HTML과 유사한 친숙한 구문으로 컴포넌트를 작성할 수 있으며, 이는 특히 중첩된 컴포넌트 구조를 표현할 때 가독성과 유지보수성을 크게 향상시킵니다.

JSX 프라그마의 변환 규칙은 사용하는 바벨 설정이나 /** @jsx */ 주석, 또는 특정 프레임워크에 따라 다르게 설정될 수 있습니다. 예를 들어, 리액트가 아닌 프레임워크에서는 createElement 함수 대신 다른 함수명을 사용할 수 있습니다.

표현식​

JSX의 강력한 기능 하나는 엘리먼트 트리 내부에서 코드를 실행하는 것입니다. 이를 위해 JSX는 중괄호 안에 자바스크립트 표현식을 포함할 수 있게 해줍니다. 중요한 점은 JSX 내부의 중괄호에는 표현식(expressions)만 허용되고 문장(statements)은 허용되지 않는다는 것입니다.

허용되는 표현식

// 산술 연산
<div>{2 + 2}</div>

// 함수 호출
<div>{formatName(user)}</div>

// 삼항 연산자
<div>{isLoggedIn ? '환영합니다' : '로그인해주세요'}</div>

허용되지 않는 문장

// if 문 
<div>{ if (condition) { return 'True' } }</div>

// for 루프 
<div>{ for (let i = 0; i < 5; i++) { ... } }</div>

문장을 사용해야 할 경우, JSX 바깥에서 로직을 처리하거나 함수로 추출한 다음 그 결과를 표현식으로 사용해야 합니다. 또는 즉시 실행 함수 표현식(IIFE)을 사용하여 중괄호 안에서 더 복잡한 로직을 실행할 수도 있습니다.

이러한 제약은 JSX의 선언적 특성을 유지하고, 컴포넌트의 가독성과 예측 가능성을 높이는 데 도움이 됩니다. 표현식만 허용함으로써 "무엇을 렌더링할지"에 집중할 수 있게 해주는 것입니다.
